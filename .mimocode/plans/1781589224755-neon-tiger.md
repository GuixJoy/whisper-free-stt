# Plan: Improve VAD, Speaker Detection, and Noise Filtering

## Problem Summary
The STT system's Voice Activity Detection (VAD) uses only RMS energy, making it unable to distinguish speech from background noise (TV, music, keyboard). This causes:
1. False triggers from non-speech sounds
2. Missed speech when speaking softly
3. Poor endpointing (VAD not cutting at right time)
4. No primary speaker identification

## Current Architecture
- `stt/vad.py`: RMS-only energy detection with window voting and hysteresis
- `stt/audio_capture.py`: mic_stream with sounddevice, no preprocessing
- `stt/transcription.py`: ASR (whisper.cpp / faster-whisper) with basic noise_reduce
- `stt/orchestrator.py`: Streaming loop feeding audio chunks to VAD → ASR → LLM

## Approach: Multi-Feature Spectral VAD

Replace RMS-only detection with a multi-feature approach that can distinguish speech patterns from noise.

### Files to Modify
1. **`stt/vad.py`** — Core changes: add spectral features to `StreamingEndpointDetector`
2. **`stt/config.py`** — Add new VAD config fields for spectral features
3. **`stt/audio_capture.py`** — Add optional audio preprocessing (AGC)
4. **`stt/orchestrator.py`** — Update calibration to compute spectral noise profile
5. **`tests/test_vad.py`** — Add tests for new spectral features

---

## Step 1: Add Spectral Feature Computation (`stt/vad.py`)

Add pure functions that compute per-chunk features:

- `compute_spectral_flux(audio, sr, n_fft, hop_length)` — High for speech onset, low for steady noise. Uses L1 distance between consecutive FFT magnitude frames.
- `compute_spectral_centroid(audio, sr, n_fft)` — Weighted mean frequency. Speech ~1-3kHz, fan/ambient noise is lower.
- `compute_zero_crossing_rate(audio)` — Fraction of sign changes. Speech ~0.03-0.25, keyboard clicks are bursty.
- `compute_band_energy_ratio(audio, sr, low, high, n_fft)` — Energy ratio in speech band (300-3400 Hz) vs total.

All functions are stateless, pure numpy (no extra dependencies).

## Step 2: Add VADConfig Fields (`stt/config.py`)

```python
# New fields in VADConfig:
use_spectral_vad: bool = True
spectral_weight: float = 0.4          # 0 = RMS only, 1 = spectral only
speech_band_low: int = 300            # Hz
speech_band_high: int = 3400          # Hz
spectral_flux_threshold: float = 0.02
```

## Step 3: Enhance StreamingEndpointDetector (`stt/vad.py`)

Add `_compute_speech_score(chunk)` that combines:
- RMS energy (existing, normalized to 0-1)
- Spectral flux (onset detection)
- Spectral centroid (speech frequency band check)
- Zero-crossing rate (voicing pattern)
- Band energy ratio (speech energy concentration)

Weighted combination with configurable `spectral_weight`. Replace the raw RMS `voiced` check with this combined score.

## Step 4: Noise Profile Calibration (`stt/orchestrator.py`)

During 1.5s calibration, also collect spectral baselines:
- Average spectral centroid, flux, ZCR, band energy ratio
- Use these to set adaptive thresholds relative to ambient noise
- Helps distinguish background noise from speech

## Step 5: Automatic Gain Control (`stt/audio_capture.py`)

Add `apply_agc(audio, target_level=0.1)` to normalize mic input levels. Limits gain to 10x max. Helps with distance/volume variation.

## Step 6: Tests (`tests/test_vad.py`)

- Test each spectral feature function with known signals
- Test `_compute_speech_score` integration
- Test AGC normalization
- Ensure existing tests pass unchanged
