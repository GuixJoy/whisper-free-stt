# UI/UX Improvement Plan — STT Desktop App

## Priority Order (User-Approved)

### Step 1: Fix Missing Modal CSS (CRITICAL)
SettingsPanel.tsx and HistoryPanel.tsx have class names with NO CSS definitions — modals are unstyled browser defaults.

**Add to `stt-ui/src/App.css`:**
- `.settings-overlay` — fixed fullscreen backdrop with blur
- `.settings-panel` — centered card with border, shadow, max-height
- `.settings-header` — flex row with title + close button
- `.settings-body` — scrollable content area
- `.settings-section` — grouped form sections with dashed borders
- `.settings-actions` — footer with save button
- `.settings-hint` — helper text style
- `.history-overlay` — fixed fullscreen backdrop with blur
- `.history-panel` — centered card with border, shadow
- `.history-header` — flex row with title + actions
- `.history-list` — scrollable card list
- `.history-card` — individual transcript card
- `.history-card-header` — mode badge + timestamp
- `.history-card-raw` — raw transcript text
- `.history-card-proc` — corrected transcript text
- `.history-card-footer` — metadata + copy button
- `.history-error` — error message banner
- `.history-empty` — empty state message

### Step 2: Extract ErrorBanner Inline Styles (HIGH)
Move 15+ inline styles from ErrorBanner.tsx to CSS classes.

**New CSS classes:**
- `.error-item` — container with border, shadow, animation
- `.error-item-header` — icon + message + dismiss flex row
- `.error-item-icon` — category icon
- `.error-item-message` — error text
- `.error-item-dismiss` — close button
- `.error-item-hint` — hint text
- `.error-item-actions` — retry button row

### Step 3: Add Full Accessibility (HIGH)
- Add `aria-label` to all icon-only buttons (✕, 📋, ⟳, ⚠)
- Add `role="dialog"` and `aria-modal="true"` to modals
- Add `autoFocus` to modal close/primary buttons
- Add keyboard Escape handler to close modals
- Add `tabIndex={0}` to interactive elements
- Ensure focus trap in modals

### Step 4: Typography Hierarchy (MEDIUM)
Establish clear weight/size scale using Caveat (400/600/700):
- h1 (app title): Caveat 700, 1.6rem
- h2 (section headers): Caveat 700, 1.2rem
- h3 (subsections): Caveat 600, 1rem
- Labels: Caveat 600, 0.88rem
- Body: Patrick Hand 400, 0.95rem
- Small/meta: Patrick Hand 400, 0.78rem

### Step 5: Standardize Spacing (LOW)
Replace raw rem/px with CSS variables throughout.

### Step 6: Focus/Hover States (MEDIUM)
Ensure all interactive elements have visible :focus-visible, :hover, :active states.
